HTML: Hyper Text Markup Language

Client and Server

Jab bhi aap kisi server ko request karenge → response me vo hamko 3 cheeze bhejega:

  • HTML
  • CSS
  • JS

Browser in tino ko samajh kar hume show karega.

Explanation

  • HTML → human body banayega
  • CSS → style dega (colour, font, page design, etc.)
  • JS → jaan dalega, interaction ke liye

History

1990s me agar kisi website par jana hota tha to poora URL likhna padta tha.

Example: https://youtube.com

Is problem ko solve karne ke liye HTML banayi gayi jisme ek page se dusre page ke link provide ho sake.

Hypertext Markup Lang

  • Hyperlink → specific word par jana
  • Formatting → bold, italic, underline

Headings

HTML provides 6 levels of headings:

  • <h1>...</h1>
  • <h2>...</h2>
  • <h3>...</h3>
  • <h4>...</h4>
  • <h5>...</h5>
  • <h6>...</h6>

Importance decreases from h1 to h6. They are block-level elements.

Default Sizes

  • h1 → 32px
  • h2 → 24px
  • h3 → 18.72px
  • h4 → 16px
  • h5 → 13.28px
  • h6 → 10.72px

Accessibility

Headings improve navigation for screen readers & create a meaningful outline.

Facts

  1. Only one <h1> per page
  2. Keywords in headings have more SEO weight

<!-- ie hamari jo website hai usme english lang. use ki
    hai then write : en
    Hindi : hi
    both : then jo lang mangey use kiya wala likh do.
    * Improve SEO nature if we write en/hi -->

<html>   <!-- HTML ROOT Element -->

<body>
    <!-- Jo content hme apni website pr dikhana hai vo yha likhenge -->
</body>

<head>
    <!-- We write here metadata ie data abt data
         ie jo kuch bhi apne body tag me likha hai uski
         info yha dete hai -->
    (embed internal CSS styles)
    <title>, <meta>, <link>, <style> <script> embed JS
    -> links external stylesheet or resources
</head>

    <!-- ie body me jo kuch bi data likha hai uska

         hisaab se hi website show kr dena. -->
